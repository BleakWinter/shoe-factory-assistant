# 接口文档

## 通用约定

- 基础地址: `http://localhost:8080`
- 响应格式: `application/json`
- 上传字段名: `file`
- PDF 预览地址格式: `/api/print-previews/{id}/preview`

## 1. 上传订单原稿

上传阶段只保存原始文件并识别订单，不生成 PDF。

```bash
curl -X POST "http://localhost:8080/api/orders/upload" \
  -F "file=@D:/test/order.xlsx"
```

支持扩展名：

- `xlsx`
- `xls`
- `png`
- `jpg`
- `jpeg`
- `webp`

Excel 上传后会解析“订单”sheet，识别订单号、客户、款号、颜色、数量、箱数、交期。图片上传后生成 `PENDING_MANUAL` 状态的订单记录。

## 2. 查询订单列表

```bash
curl "http://localhost:8080/api/orders?page=1&size=20"
```

过滤参数：

- `orderNo`
- `styleNo`
- `customerName`
- `deliveryDate`，格式 `yyyy-MM-dd`
- `recognitionStatus`，可选 `RECOGNIZED`、`PENDING_MANUAL`、`FAILED`

示例：

```bash
curl "http://localhost:8080/api/orders?orderNo=PO123&styleNo=A88&recognitionStatus=RECOGNIZED&page=1&size=20"
```

## 3. 按打印类型生成 PDF 预览

选择打印类型后才生成 PDF。

```bash
curl -X POST "http://localhost:8080/api/orders/1/print-previews" \
  -H "Content-Type: application/json" \
  -d "{\"printType\":\"ORDER\"}"
```

打印类型：

- `ORDER`: 提取 Excel 中“订单”sheet，设置 A4、横向、一页宽、有效打印区域和小页边距后转 PDF。
- `PACKING`: 提取 Excel 中“装箱单”sheet，设置同样打印格式后转 PDF。

返回示例：

```json
{
  "success": true,
  "message": "OK",
  "data": {
    "id": 1,
    "previewNo": "PV20260511103000123A1B2C3D4",
    "orderId": 1,
    "orderNo": "PO123",
    "printType": "ORDER",
    "previewUrl": "/api/print-previews/1/preview",
    "status": "READY"
  }
}
```

## 4. 打开 PDF 预览

```text
http://localhost:8080/api/print-previews/1/preview
```

## 5. 确认打印并创建打印任务

```bash
curl -X POST "http://localhost:8080/api/print-tasks" \
  -H "Content-Type: application/json" \
  -d "{\"previewId\":1,\"printerName\":\"Office Printer\",\"copies\":2,\"priority\":0}"
```

第一阶段只创建任务，不调用真实打印机。

## 6. 查询待打印任务

本地 `print-agent` 后续轮询该接口：

```bash
curl "http://localhost:8080/api/print-tasks/pending?limit=20"
```

## 7. 更新打印任务状态

```bash
curl -X PATCH "http://localhost:8080/api/print-tasks/1/status" \
  -H "Content-Type: application/json" \
  -d "{\"status\":\"PRINTING\"}"
```

可用状态：

- `PENDING`
- `PRINTING`
- `SUCCESS`
- `FAILED`
- `CANCELED`
