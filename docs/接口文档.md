# 接口文档

## 通用约定

- 基础地址: `http://localhost:8081`
- 响应格式: `application/json`
- 上传字段名: `file`
- PDF 文件读取地址: `/api/print-tasks/{taskId}/pdf`

## 1. 上传订单 Excel

上传阶段保存原始 Excel，并读取订单基础信息写入 `order_record`，不立即生成 PDF，也不完整落订单明细。

```bash
curl -X POST "http://localhost:8081/api/orders/upload" \
  -F "file=@D:/test/order.xlsx"
```

支持扩展名：

- `xlsx`
- `xls`

上传阶段落库关系：

- `order_record`: 订单流水号、客户、开发编号汇总、总对数、总箱数。
- `order_sheet_print_task`: 订单/装箱单两条打印任务，并保存原始文件名和原稿本地路径。

订单明细和装箱单明细需要用户后续点击识别按钮后再落库。

## 2. 查询订单主表

```bash
curl "http://localhost:8081/api/orders?page=1&size=20"
```

过滤参数：

- `orderNo`
- `developmentNos`
- `recognitionStatus`，可传 `ORDER_PENDING`、`PACKING_PENDING`、`ORDER_RECOGNIZED`、`PACKING_RECOGNIZED`、`FAILED`

示例：

```bash
curl "http://localhost:8081/api/orders?orderNo=26082&developmentNos=JCD&page=1&size=20"
```

## 3. 查询订单详情

详情接口返回 `order_record_detail`，每条明细里会带 `processes` 数组，对应 `order_detail_process`。

```bash
curl "http://localhost:8081/api/orders/1/details"
```

## 4. 手动识别

上传后不会落订单明细或装箱单明细，用户点击按钮后再调用：
```bash
curl -X POST "http://localhost:8081/api/orders/1/recognize-order"
curl -X POST "http://localhost:8081/api/orders/1/recognize-packing"
```

## 5. 明细图片

```text
http://localhost:8081/api/orders/details/{detailId}/image
http://localhost:8081/api/orders/packing-details/{detailId}/image
```

## 6. 生成 PDF 预览

打印任务已经自带打印类型，点击某一条任务后才生成 PDF。当前版本生成成功后，PDF 文件会保存到 `D:/清化资料/pdf/年份/月/订单号`，同时回写到 `order_sheet_print_task.preview_pdf_path`。

```bash
curl -X POST "http://localhost:8081/api/print-tasks/1/preview"
curl -X POST "http://localhost:8081/api/print-tasks/1/preview/regenerate"
```

当前打印类型：

- `ORDER`: 订单 PDF，回写 `order_sheet_print_task.preview_pdf_path`。
- `PACKING`: 装箱单 PDF，回写 `order_sheet_print_task.preview_pdf_path`。

生成失败时，任务会更新为：

- `status = FAILED`
- `error_message = 失败原因`

返回示例：

```json
{
  "success": true,
  "message": "OK",
  "data": {
    "id": 1768280000000,
    "previewNo": "PV20260513143000123A1B2C3D4",
    "orderId": 1,
    "orderNo": "26082",
    "printType": "ORDER",
    "previewUrl": "/api/print-tasks/1/pdf",
    "status": "READY"
  }
}
```

## 7. 打开 PDF

```text
http://localhost:8081/api/print-tasks/1/pdf
```

## 8. 打印列表

打印列表查询 `order_sheet_print_task`，接口名保留为：

```bash
curl "http://localhost:8081/api/print-tasks"
curl "http://localhost:8081/api/print-tasks/pending?limit=20"
```

- 上传 Excel 后创建订单、装箱单两个整单级打印任务。
- 外箱贴标、内盒贴标、标签等后续单独建表。
- 详细设计见 `docs/打印任务与识别流程设计.md`。

标记已打印：

```bash
curl -X PATCH "http://localhost:8081/api/print-tasks/1/printed"
```

手动更新状态：

```bash
curl -X PATCH "http://localhost:8081/api/print-tasks/1/status" \
  -H "Content-Type: application/json" \
  -d "{\"status\":\"FAILED\",\"errorMessage\":\"打印机未连接\"}"
```

## 9. 开发编号配置

盒规/重量配置：

```bash
curl "http://localhost:8081/api/style-configs?page=1&size=20"
curl -X PUT "http://localhost:8081/api/style-configs/1" \
  -H "Content-Type: application/json" \
  -d "{\"boxSpec\":\"30x20x10\",\"netWeightPerPair\":0.8,\"grossWeightPerPair\":1.0}"
```

鞋价配置：

```bash
curl "http://localhost:8081/api/price-configs?page=1&size=20"
curl -X PUT "http://localhost:8081/api/price-configs/1" \
  -H "Content-Type: application/json" \
  -d "{\"shoePrice\":128.00}"
```

两个列表都支持 `developmentNos`、`incompleteOnly`、`page`、`size` 查询参数。
