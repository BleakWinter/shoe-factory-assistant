# 接口文档

## 通用约定

- 基础地址: `http://localhost:8080`
- 响应格式: `application/json`
- 上传字段名: `file`
- PDF 文件读取地址: `/api/orders/{orderId}/pdf/{printType}`

## 1. 上传订单 Excel

上传阶段保存原始 Excel 并解析订单主表和明细，不立即生成 PDF。

```bash
curl -X POST "http://localhost:8080/api/orders/upload" \
  -F "file=@D:/test/order.xlsx"
```

支持扩展名：

- `xlsx`
- `xls`

落库关系：

- `order_record`: 订单流水号、客户、开发编号汇总、总对数、总箱数、原稿路径、PDF 路径。
- `order_record_detail`: 楦头、开发编号、客人订单号、PO、材质、尺码数量等明细。
- `order_detail_process`: 每条明细的处理状态。

## 2. 查询订单主表

```bash
curl "http://localhost:8080/api/orders?page=1&size=20"
```

过滤参数：

- `orderNo`
- `customerName`
- `developmentNo`
- `recognitionStatus`，可传 `0/1/2/3` 或 `PENDING/RECOGNIZED/PENDING_MANUAL/FAILED`

示例：

```bash
curl "http://localhost:8080/api/orders?orderNo=26082&developmentNo=JCD&page=1&size=20"
```

## 3. 查询订单详情

详情接口返回 `order_record_detail`，每条明细里会带 `processes` 数组，对应 `order_detail_process`。

```bash
curl "http://localhost:8080/api/orders/1/details"
```

## 4. 明细图片

```text
http://localhost:8080/api/orders/details/{detailId}/image
```

## 5. 生成 PDF 预览

选择打印类型后才生成 PDF。生成成功后，PDF 文件会保存到 `D:/清化资料/pdf`，同时回写到 `order_record`。

```bash
curl -X POST "http://localhost:8080/api/print-tasks/1/preview" \
  -H "Content-Type: application/json" \
  -d "{\"printType\":\"ORDER\"}"
```

打印类型：

- `ORDER`: 订单 PDF，回写 `order_record.order_pdf_path`。
- `PACKING`: 装箱单 PDF，回写 `order_record.packing_pdf_path`。

返回示例：

```json
{
  "success": true,
  "message": "OK",
  "data": {
    "id": 1768280000000,
    "previewNo": "PV20260513143000123A1B2C3D4",
    "orderId": 1,
    "orderNo": "26082",
    "printType": "ORDER",
    "previewUrl": "/api/orders/1/pdf/ORDER",
    "status": "READY"
  }
}
```

## 6. 打开 PDF

```text
http://localhost:8080/api/orders/1/pdf/ORDER
http://localhost:8080/api/orders/1/pdf/PACKING
```

## 7. 打印列表

打印列表当前直接展示 `order_record`，接口名保留为：

```bash
curl "http://localhost:8080/api/print-tasks"
```

`id` 和 `orderId` 都是 `order_record.id`。
