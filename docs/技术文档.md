# 技术文档

## 项目概览

`shoe-factory-assistant` 当前阶段实现订单 Excel 上传、订单主表/明细解析、订单详情查看、按打印类型生成 PDF 预览。

## 模块划分

- `backend`: 负责原稿保存、订单解析、订单查询、PDF 生成和 PDF 文件读取。
- `web`: 负责订单列表、订单详情、订单上传、打印类型选择和 PDF 预览。
- `print-agent`: 第二阶段负责连接本地打印设备并执行真实打印。
- `sql`: 负责数据库初始化和表结构脚本。

## 技术约定

- 后端使用 Java 17、Spring Boot 3、MyBatis-Plus、MySQL 8。
- Excel 读取和打印设置使用 Apache POI。
- Excel 转 PDF 优先通过 LibreOffice 命令行完成，Windows 上可用 Microsoft Excel COM 兜底。
- 前端使用 React、Vite、TypeScript、Ant Design。

## 核心数据流

1. 上传 Excel 原稿，文件保存到 `D:/清化资料/年份/客户/月`。
2. `order_record` 保存订单主表汇总、原稿路径、PDF 路径和打印状态。
3. `order_record_detail` 保存 Excel “订单”sheet 里的每一行明细。
4. `order_detail_process` 保存明细对应的订包装、定大底、定中底等处理状态。
5. `/orders` 页面展示 `order_record`；点击详情后展示 `order_record_detail` 和 `order_detail_process`。
6. 点击打印并选择 `ORDER` 或 `PACKING` 后，后端生成 PDF 到 `D:/清化资料/pdf`，并把路径回写到 `order_record.order_pdf_path` 或 `order_record.packing_pdf_path`。
