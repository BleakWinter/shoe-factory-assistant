# 部署文档

## 环境要求

- JDK 17+
- Maven 3.8+
- Node.js 18+
- MySQL 8+
- LibreOffice

注意：本机如果同时装了 Java 8 和 Java 17，必须确认 Maven 使用的是 Java 17。否则会在 `switch` 表达式、text block、record 等 Java 17 语法处报大量编译错误。

```powershell
$env:JAVA_HOME="D:\develop\JAVA\jdk-17.0.18"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
java -version
mvn -v
```

## 部署步骤

1. 执行 `sql/init.sql` 初始化数据库。
2. 执行 `sql/print_table.sql` 创建打印任务表。
3. 旧库升级时按需要执行 `sql/add_order_sheet_print_task.sql`、`sql/move_original_file_to_sheet_print_task.sql`、`sql/add_shipping_note_task.sql`、`sql/add_component_order_task.sql`。
4. 按目标环境修改 `backend/src/main/resources/application-dev.yml` 或 `backend/src/main/resources/application-prod.yml` 中的数据库连接。
5. 确认 `soffice` 可在命令行执行，或配置 `app.file-storage.libre-office-command` 为完整路径。
6. 构建并启动 `backend`。
7. 启动 `web` 进入订单工作台。
8. 第一阶段暂不需要启动 `print-agent`，只需要通过接口创建和更新打印任务。

## 后端构建

后端按 Spring Profile 分为 `dev` 和 `prod`：

- `dev`：默认环境，读取 `application-dev.yml`，端口为 `8081`。
- `prod`：生产环境，读取 `application-prod.yml`，端口为 `8080`。

开发环境启动：

```powershell
cd D:\work\shoe-factory-assistant\backend
$env:JAVA_HOME="D:\develop\JAVA\jdk-17.0.18"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn clean package -Pdev
java -jar target\shoe-factory-backend-0.1.0-SNAPSHOT.jar
```

生产环境构建并启动：

```powershell
cd D:\work\shoe-factory-assistant\backend
$env:JAVA_HOME="D:\develop\JAVA\jdk-17.0.18"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn clean package -Pprod
java -jar target\shoe-factory-backend-0.1.0-SNAPSHOT.jar --spring.profiles.active=prod
```

也可以在启动 jar 时显式指定环境：

```powershell
java -jar target\shoe-factory-backend-0.1.0-SNAPSHOT.jar --spring.profiles.active=prod
```

只做编译检查：

```powershell
cd D:\work\shoe-factory-assistant\backend
$env:JAVA_HOME="D:\develop\JAVA\jdk-17.0.18"
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn -DskipTests compile
```

## 前端启动

```powershell
cd D:\work\shoe-factory-assistant\web
npm.cmd install
npm.cmd run dev
```

Vite 会把 `/api` 代理到本地 dev 后端 `http://localhost:8081`。
