# 订包装业务规则

## 1. 数据来源

订包装单使用订单对应的装箱单明细生成。系统应先将订单明细与 `order_packing_detail` 匹配，再按匹配到的装箱明细汇总包装数量。

## 2. 装箱明细与“几装”的含义

- 一行装箱明细代表一个箱子。
- “14装、13装、12装……2装”表示一个箱子能够装多少双鞋，不是鞋子的尺码。
- 每个“几装”栏填写的是采用这种装法的箱子数量。
- 一个箱子的装载对数等于该行装箱明细中各鞋码数量之和。

计算方式：

```text
单箱装载对数 = Σ（该行各鞋码数量）
某规格箱数 = 单箱装载对数等于该规格的装箱明细行数
总数量 = Σ（单箱装载对数 × 对应箱数）
```

例如三行装箱明细的单箱装载对数分别为 `12、12、10`，则订包装单汇总为：

```text
12装：2箱
10装：1箱
总数量：12 × 2 + 10 × 1 = 34双
```

装载对数不在模板 `2装～14装` 范围内时，不应强行归入其他栏位，应提示异常并由用户确认。

## 3. 订包装任务快照

订包装任务保存时，应保留原订单明细和装箱明细的关联 ID，同时保存生成下单单据所需的快照。历史任务和已导出的订包装单以快照为准，后续修改订单或装箱单不能改变历史下单内容。

建议快照至少包括：

- 订单号、开发款号、总数量。
- 内盒规格、胶袋规格、布袋规格、备注。
- `2装～14装` 各规格的箱数汇总。
- 款式图片或图片引用。
- 下单日期及来源订单明细、装箱明细 ID。

## 4. Excel 模板字段映射

`包装.xlsx` 的“彩盒”工作表按以下方式生成：

| 模板字段 | 数据含义 |
| --- | --- |
| 日期 | 订包装任务的下单日期 |
| 订单号 | 来源订单流水号 |
| 开发款号 | 来源订单明细的开发编号 |
| 总数量 | 所有关联装箱明细行的单箱装载对数之和 |
| 内盒规格 | 该开发款号对应的内盒规格 |
| 14装～2装 | 对应单箱装载对数的装箱明细行数 |
| 胶袋规格 | 该开发款号对应的胶袋规格 |
| 布袋规格 | 该开发款号对应的布袋规格 |
| 备注 | 订包装备注 |
| 图片 | 款式图片 |
